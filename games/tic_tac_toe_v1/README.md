# tic_tac_toe_v1

Крестики-нолики для Pixel Battle в формате 1v1.

## Что делает игра

- Игровое поле рендерится как квадрат `15x15` внутри арены `24x15`.
- Поле делится на `3x3` зоны по `5x5` пикселей.
- Символы рисуются масками `5x5`:
  - `X` для игрока 1
  - `O` для игрока 2
- Рамка поля пульсирует цветом текущего игрока.
- После победы в матче:
  - вся арена заливается цветом победителя,
  - по центру рисуется `WIN`,
  - через задержку матч автоматически перезапускается.

## Файлы игры

- `tic_tac_toe_v1.lua` — вся логика и рендер игры.
- `game.json` — параметры комнаты/поля от разработчика.
- `config.json` — настраиваемые параметры раунда и матча.

## Основные параметры config.json

- `Bright` — базовая яркость.
- `RoundWinTarget` — до скольких побед играется матч.
- `TurnTimeSec` — время на ход в секундах.
- `RoundStartCountdown` — обратный отсчет перед стартом.
- `RoundEndDelayMs` — пауза после конца раунда.
- `RestartDelayMs` — длительность экрана `WIN` перед рестартом.
- `HoverMinWeight` — минимальный вес нажатия для фиксации хода.

## Состояния игры

- `GAMESTATE_SETUP` — предстартовый отсчет.
- `GAMESTATE_GAME` — активный раунд.
- `GAMESTATE_POSTGAME` — экран победы матча (`WIN`) и ожидание рестарта.

Поток:

`SETUP -> GAME -> (POSTGAME при победе матча) -> SETUP/GAME`

## Контракт с движком

Скрипт реализует все обязательные методы:

- `StartGame`
- `NextTick`
- `RangeFloor`
- `GetStats`
- `PauseGame` / `ResumeGame`
- `SwitchStage`
- `PixelClick` / `ButtonClick`
- `DefectPixel` / `DefectButton`

## Коротко о рендере

- Игровой рендер: `drawBoard()`
- Экран победы: `drawWinScreen()`
- Сброс матча: `resetMatch()`

Это сделано так, чтобы рендер и стейт-машина были разделены и код легче поддерживался.
